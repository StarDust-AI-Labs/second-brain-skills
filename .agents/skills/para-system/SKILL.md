# PARA组织系统 (Projects, Areas, Resources, Archives)

## R — 原文 (Reading)

> "我将这种组织方法命名为PARA系统，它很好地概括了生活中常见的四大信息类型，也即项目（Project）、领域（Area）、资源（Resource）以及存档（Archive）。这四大类型是普遍适用的，足以囊括信息的各种种类、渠道、格式以及目的。"
>
> —— 蒂亚戈·福特，《打造第二大脑》第五章

> "相比基于'信息来源'的组织方式，我更加推荐以'信息归宿'为标尺——更确切地说，就是以该信息可能促成的结果为导向。知识的价值并不体现在它的组织和分类形式上，而是体现在它是否能对与你相关的人或事产生影响。"
>
> —— 《打造第二大脑》第五章

## I — 方法论骨架 (Interpretation)

PARA不是一个文件夹系统——它是一个**生产系统**。它不关心"这条信息是什么"，只关心"这条信息能帮我做成什么"。

四个分类按照"可行动性"从高到低排列：

**P — 项目 (Projects)**：你正在积极执行、有明确终点和交付物的事项。例如"Q3市场分析报告""下个月的旅行计划""给团队做的一次培训"。一个健康的同时活跃项目数是5-15个。信息在"项目"中因为你马上要用——它离"行动"最近。

**A — 领域 (Areas)**：你需要长期负责、没有明确终点的事项。例如"健康""财务管理""团队建设""职业发展"。与项目的关键区别：项目有终点（"完成报告"），领域永无止境（"保持健康"）。每个领域都有你自定的标准，如"每月储蓄收入的20%"。

**R — 资源 (Resources)**：你感兴趣的、未来可能有用的话题。例如"想学的日语""关注的室内设计""收集的食谱"。资源中的信息处于"休眠但可用"状态——你不急于使用，但希望未来需要时能找到。

**A — 存档 (Archives)**：从上述三类中退下来的休眠信息。已完成的项目、不再关注的领域、放弃的兴趣。关键操作：归档而非删除——数字空间不怕存，只怕乱。

**分类速查规则**：拿到一条新信息时，从上往下问——
1. 能帮助某个正在进行的**项目**吗？→放项目
2. 不能的话，跟某个长期**领域**相关吗？→放领域
3. 是感兴趣但暂时没用的**资源**吗？→放资源
4. 以上都不是？→放存档

**核心哲学**：信息可以在四类之间自由流动。一条笔记今天在"项目"中助力行动，明天可能流转到"存档"休眠，后天又因新项目重新激活。PARA随你的生活变化而变化——它不是一成不变的分类表。

## A1 — 书中的应用 (Past Application)

### 案例 1：作者自己的PARA系统
- **问题**：作者积累数千份笔记后，尝试过主题分类、杜威十进分类法等多种组织方式，全部失败——因为分类本身消耗的时间超过了使用笔记的时间。
- **方法论的使用**：在数字笔记应用中创建四个顶级文件夹(P/A/R/A)；每个活跃项目有一个子文件夹；所有笔记最终只经过三层结构到达具体内容。
- **结果**：维护成本降到极低——每周只需数分钟即可完成信息归类。作者可以同时管理5-15个活跃项目而不感到混乱。

### 案例 2：学员Amelia的观念转变
- **问题**：学员Amelia认为互联网是"哗众取宠和充满戾气"的地方，对数字世界极度抵触。
- **方法论的使用**：引入PARA系统后，她开始有选择性地抓取和分类信息——只把对教练工作有用的内容放进对应"领域"文件夹。
- **结果**：从"全面排斥互联网"转变为"择其善者而从之"，这为她的领导力教练事业开辟了新天地。

### 案例 3：特怀拉·萨普的文件盒（历史类比）
- **问题**：著名舞蹈编导特怀拉·萨普每次开启新项目时需要处理大量碎片化素材（笔记本、新闻剪报、CD、录像带、照片等）。
- **方法论的使用**：她使用物理"文件盒"——启动新项目时拿出一个盒子，贴上项目名标签，把一切相关素材放进去。
- **结果**：文件盒成为她的"力量之源"，让她敢于接受一切挑战——"文件盒让我和项目之间产生了紧密的羁绊"。这与PARA的"项目文件夹"概念完全一致。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 电脑桌面堆满文件、下载文件夹有几百个未分类的文件
2. 笔记应用中有大量笔记但不知道如何建立文件夹结构
3. 每次保存内容时纠结"该放哪个文件夹"
4. 正在从旧笔记系统迁移到新系统，想建立更科学的结构
5. 多个设备（电脑+手机+平板）上的文件组织方式不一致

### 语言信号 (用户的话里出现这些就应激活)

- "我的文件夹太乱了"
- "不知道怎么分类整理"
- "笔记该放在哪里"
- "下载文件夹快爆了"
- "如何组织我的数字文件"
- "文件管理一团糟"
- "为什么我的分类系统不管用"

### 与相邻 skill 的区分

- 与 `second-brain-code` 的区别：PARA是CODE中"组织"环节的具体落地，CODE是整体路线图
- 与 `intermediate-packets` 的区别：PARA管"放在哪"，半熟素材管"怎么用"
- 与 `knowledge-lifecycle` 的区别：PARA是结构框架，知识生命周期是维护流程（项目清单、周月小结）

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **清点当前状态**
   - 询问用户：你主要在哪些平台上存储信息？（笔记应用/电脑文件夹/云盘/浏览器书签）
   - 询问：你现在有多少正在进行的项目？（工作+个人）
   - 完成标准：了解用户的信息分布和项目数量

2. **创建PARA四分类骨架**
   - 引导用户在主要平台上创建四个顶层文件夹/分类：`项目` `领域` `资源` `存档`
   - 完成标准：四个顶级分类已创建
   - 重要提示：不要建议一次性整理所有历史文件——那会让人放弃。先将所有历史文件统一放入"存档"文件夹（标注日期），从今天开始的新内容用PARA分类。

3. **识别活跃项目并建立子文件夹**
   - 引导用户列出所有正在进行的事项（工作+个人），为每个事项在"项目"下创建子文件夹
   - 预期数量：5-15个
   - 完成标准：至少为3个最重要的活跃项目创建了文件夹
   - 判停条件：如果用户列不出任何项目→先帮用户识别什么是"项目"（有明确终点和交付物的事项）

4. **识别责任领域并建立子文件夹**
   - 引导用户列出需要长期负责的领域（健康、财务、职业、家庭等），创建对应的领域子文件夹
   - 完成标准：至少识别了3个核心领域并创建了文件夹

5. **立即试验：存一条笔记**
   - 让用户找一条最近保存但还没归类的内容，用PARA分类速查表（P→A→R→A顺序）判断该放哪里
   - 完成标准：用户亲手完成了一次分类决策

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 团队共享知识库或公司Wiki——PARA是为个人设计的，团队需要协作编辑和权限管理功能
- 用户已有一套稳定运行的组织系统且不想改变——不要为了PARA而PARA
- 处理需要严格合规或法律要求的结构化文档（如合同、财务报表）——这些应使用专门的文档管理系统

### 作者在书中警告的失败模式

- **按主题而非按行动分类**：这是最核心的陷阱。把所有读书笔记放一起只因为它们都来自书，会导致信息与使用场景脱节——这是"信息坟墓"的根源。
- **组织完美主义**：沉迷于优化文件夹结构和标签体系，把系统维护本身当成了目的，却从未基于这些信息创造任何价值。
- **试图初次就做到完美**：PARA是动态系统，分类会随着你的生活变化而变化。不要花数小时纠结"每条笔记的完美位置"。

### 作者的盲点 / 时代局限

- 偏向个人使用场景，团队协作中的信息共享、权限管理、版本冲突等问题未深入讨论
- 假设用户具备基本的数字文件管理能力，对于连"文件夹"概念都不熟悉的用户缺少入门引导

### 容易混淆的邻近方法论

- 杜威十进分类法/图书馆分类法：为"全面检索"设计，PARA为"快速行动"设计。前者追求完整性和客观性，后者追求实用性和个人化。
- 标签系统(Tags)：标签和PARA文件夹不是互斥的。文件夹提供"主路径"，标签提供"交叉引用"。作者不推荐标签作为主系统。

## 合理化防御（Prebuttal）

Agent 在压力下会为跳过 PARA 分类找到合理理由。以下是预先反驳：

| Agent 的借口 | 预先反驳 |
|-------------|---------|
| "这篇文章明显是技术类，放资源就好" | 分类不是看内容标签——是看"它帮你推进什么"。同一个技术文章，对项目是参考，对领域是知识积累 |
| "用户没提到具体项目，放收件箱就好" | 收件箱是暂存区，不是万能桶。应该追问归属而非默认为收件箱 |
| "分类太麻烦，用户自己会整理" | 你的核心价值就是替用户整理。跳过分类 = 把最该做自动化的事还给用户 |
| "这个是存档，直接放存档就好" | PARA 是自底向上筛选的——先看是不是项目（P），再考虑领域（A），最后才是资源/存档。不要反着来 |
| "字段都差不多，少一个无所谓" | PARA 的四个分类决定了笔记的检索路径。放错一个 = 永远找不到它 |
