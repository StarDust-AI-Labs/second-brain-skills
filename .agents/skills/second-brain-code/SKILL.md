# CODE信管法则 (Capture, Organize, Distill, Express)

## R — 原文 (Reading)

> "为了帮助大家构建起属于自己的第二大脑，我发明了一个简单而直观的四步法，也即所谓的'信管法则（CODE）'——抓取（Capture）、组织（Organize）、提炼（Distill）、表达（Express）。"
>
> —— 蒂亚戈·福特，《打造第二大脑》第三章

> "信管法则的前两个步骤——抓取和组织，构成了'发散'过程。这一阶段的主题是研究、探索和萌发创意。后面两个步骤——提炼和表达，则构成了'聚合'过程。这时，我们停止吸纳新思想，并利用收集完毕的知识模块创建新事物。"
>
> —— 《打造第二大脑》第八章

---

## I — 方法论骨架 (Interpretation)

CODE是一套将"信息消费"转化为"知识创造"的四个顺序阶段：

**阶段1 — 抓取 (Capture)**：你不是信息管道——你是信息策展人。不要无差别地保存一切，只抓取那些触动内心、让你"眼前一亮"的内容。抓取时不要同时思考分类和用途，快速存入"收件箱"即可。

**阶段2 — 组织 (Organize)**：不是按主题（"心理学""科技"）分类，而是按"这条信息能帮助我推进哪个项目？"来归类。使用PARA系统（项目/领域/资源/存档）——信息离你的行动越近，位置越靠前。

**阶段3 — 提炼 (Distill)**：逐层标记重点——先加粗核心句，再高亮精华，最后用自己话写纲要。你的目标是让未来的自己能在30秒内判断这条笔记是否有用。只在有明确创作目标时才深度提炼。

**阶段4 — 表达 (Express)**：信息管理的终点不是"知道"，而是"做出"。把你积累的半熟素材拼装成文章、演示、方案、决策。越早分享未完成的作品，越早获得反馈——反馈是知识的加速器。

四个阶段并非一次性的线性流程，而是循环往复：每一次表达都会触发新信息的抓取需求，形成无限迭代。

**关键洞察**：前两步（抓取+组织）属于"发散"——心态要开放，不怕多；后两步（提炼+表达）属于"聚合"——心态要果断，敢于砍。大多数人卡在"永远在发散，从未到聚合"的困境中。

## A1 — 书中的应用 (Past Application)

### 案例 1：作者的个人知识管理转型
- **问题**：作者患功能性发声障碍后，患有严重短期记忆丧失，24岁的大脑机能仅与8岁孩童相当。
- **方法论的使用**：他开始系统性记录病史，将所有医疗信息数字化——抓取诊断、检查结果、治疗方案；按症状/药物/医生组织信息；提炼关键规律（发现止痛药掩盖了真实病因）；最终通过改变生活方式治愈了自己。
- **结论**：笔记系统在缓解病情方面所发挥的作用不亚于任何药物或治疗手段。
- **结果**：从一个成绩平平的学生变成全优毕业生，在硅谷咨询公司成为"关键先生"，最终创立了全球性的第二大脑课程。

### 案例 2：从分析师到知识管理布道者
- **问题**：作者进入硅谷咨询公司后，面对每天数百封邮件、数十条消息的信息轰炸，几乎崩溃。
- **方法论的使用**：将所有会议、通话、调研笔记数字化；按项目组织信息；提炼关键洞察以备客户演示；通过分享笔记成为团队中唯一掌握所有重要细节的人。
- **结果**：同事们称他为"大神"，任何急需的文档、案例、客户意见他都能秒查。这直接催生了后来"打造第二大脑"在线课程。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 刚入职/转行/开启新领域学习，面对海量新知识不知从何消化
2. 感觉自己在大量阅读、听课、收藏，但实际产出和工作能力没有提升
3. 笔记散落在多个应用（备忘录、便签、云盘、纸质笔记本），找一个东西要翻遍所有角落
4. 想要建立一套"终身学习"的个人知识体系，但不确定从哪开始
5. 每次启动新项目都感觉"从零开始"，明明记得以前做过类似的东西但找不到

### 语言信号 (用户的话里出现这些就应激活)

- "信息太多不知道怎么整理"
- "学了很多但用不出来"
- "笔记散落各处找不到"
- "需要一套知识管理系统"
- "如何构建第二大脑"

### 与相邻 skill 的区分

- 与 `para-system` 的区别：CODE是顶层路线图（四步全流程），PARA是其中"组织"环节的具体分类系统
- 与 `progressive-summarization` 的区别：CODE是总框架，渐进式归纳法是其中"提炼"环节的具体技术
- 与 `capture-criteria` 的区别：CODE定义流程，抓取标准定义"组织"环节中"什么值得抓"的判断准则

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **诊断用户当前状态**
   - 询问：你现在最困扰的是哪一步？是不知道记什么（抓取）？记了找不到（组织）？找到了但太长看不完（提炼）？还是积累了却没产出（表达）？
   - 完成标准：明确用户卡在CODE的哪个环节

2. **根据诊断结果，引导用户走一遍CODE四步**
   - 若卡在抓取→推荐设置一个数字"收件箱"(笔记应用的默认文件夹)，告知"只记让你心动的，其他省掉"
   - 若卡在组织→推荐建立PARA四个顶级文件夹，把收件箱内容优先分配到"项目"文件夹
   - 若卡在提炼→推荐渐进式归纳法：对一条重要笔记花5分钟加粗关键句，标记高亮
   - 若卡在表达→推荐思想群岛：列出要创作的内容大纲，从第二大脑拖拽相关笔记作为素材
   - 完成标准：用户走通了至少一个之前卡住的环节并产出了一个具体成果（如整理了一个项目文件夹）
   - 判停条件：若用户只想了解框架→跳到步骤3

3. **给出最小可行的下一步行动**
   - 不要建议"重新整理所有历史笔记"——那会让人放弃
   - 建议："选一个你现在正在做的小项目（如'下周五的会议准备'），用CODE四步跑一遍：记录初始想法→为它建一个文件夹→把相关笔记移进去→产生一个会议大纲。只需20分钟。"
   - 完成标准：用户有了一个明确的、20分钟内可完成的行动项

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户只是想了解某款具体笔记软件的功能对比——这是工具选择问题，CODE是方法论，不依赖特定工具
- 用户已经有成熟的知识管理习惯且高效运转中——不要为了用而用
- 纯学术研究场景中要求完整的引文管理体系（如LaTeX+BibTeX）——CODE偏重"行动产出"而非"学术规范"

### 作者在书中警告的失败模式

- **抓取瞬间同步分类**：试图在记录笔记的同时确定其存放位置和用途，这会增加心理负担并降低记录积极性。必须将抓取和组织分为两个独立步骤。
- **把组织系统本身当作目的**：沉迷于优化文件夹结构和标签体系，却从未产出任何作品。

### 作者的盲点 / 时代局限

- 依赖稳定的数字设备环境，对数字素养低或不具备智能手机/电脑条件的人群缺乏替代方案
- 偏向硅谷知识工作者的工作节奏，996工作者可能难以找到维护系统的碎片时间
- "记录→创造力"的因果关系未经严格验证

### 容易混淆的邻近方法论

- GTD (Getting Things Done)：GTD管理的是"任务和行动"，CODE管理的是"信息和知识"。两者互补——GTD告诉你做什么，CODE告诉你用什么做。
- Zettelkasten (卡片盒笔记法)：Zettelkasten强调原子化笔记之间的链接网络，CODE是一套更外层的工作流程。两者可以共存。
