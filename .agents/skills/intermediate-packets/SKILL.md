# 半熟素材思维 (Intermediate Packets)

## R — 原文 (Reading)

> "这里所说的系统即指我们的第二大脑。至于其中收录的所有零碎信息的集合，我愿称之为'半熟素材'，也即构建最终作品所必需的、个别的、具体的创作元素。例如小组会议的纪要、相关研究成果的列表、与小伙伴们开展的头脑风暴、用于市场分析的幻灯片、电话会议确定的行动项目等。任何一则笔记都有可能在某个更高级别的项目或目标中扮演'半熟素材'的角色。"
>
> —— 蒂亚戈·福特，《打造第二大脑》第七章

> "'从零开始''白纸作画'等概念将会从你的创作字典里彻底消失——为什么不将新的创作建立在历史经验所积累的宝贵财富之上呢？"
>
> —— 《打造第二大脑》第七章

## I — 方法论骨架 (Interpretation)

半熟素材是一种**将工作产出资产化**的思维模式。它的核心洞见是：你今天创造的一切——笔记、幻灯片、邮件模板、代码片段、会议记录——都不是一次性消耗品，而是可以跨项目、跨时间复用的知识资产。

**五种常见的半熟素材类型**：
1. **干货笔记**：对书籍/文章深度提炼后的精要梗概（渐进式归纳法的产物）
2. **落选素材**：被当前项目砍掉但未来可能有用的内容（压缩范围的副产品）
3. **过程性文件**：项目中的中间产物——草图、大纲、计划、会议记录
4. **最终成果**：已交付的项目产出，可以直接用于类似项目
5. **他山之石**：他人创造的可借鉴资产（需标明出处并取得许可）

**80%完成原则**：当你已经为某个主题积累了足够多的半熟素材时，项目其实已经完成了80%的前期工作。真正的"创作"只是将已有的知识模块进行排列组合和打磨。这也是作者"只启动那些已经完成了80%的项目"这一格言的由来。

**核心转变**：从"每次干完活就把过程扔掉"到"每次干完活都从中提取可复用的模块"。这不是额外的负担——只是养成为产出物多花5分钟标注和保存的习惯。

## A1 — 书中的应用 (Past Application)

### 案例 1：作者的客户咨询工作
- **问题**：作者在硅谷咨询公司工作时，为顶级客户提供咨询服务。每次项目结束后，辛苦积累的研究成果、分析框架、演示材料都随项目结束而被丢弃。
- **方法论的使用**：他将每次项目产出的演示幻灯片、分析模板、客户反馈要点等都保存为半熟素材。新项目启动时，先搜索这些素材再开始工作。
- **结果**：同事们称他为"大神"——因为他总是能翻出别人急需的文档、提供孜孜以求的案例、复述某位客户三周前发表的意见。半熟素材积累让他成为团队中的"知识中枢"。

### 案例 2：学员Rebecca的教学准备
- **问题**：佛罗里达州某大学的教育心理学教授Rebecca，每次准备讲演都需要大段整块时间来整理思路，但作为忙碌的母亲和专业工作者，整块时间越来越稀缺。
- **方法论的使用**：她用碎片时间创建短小精悍的笔记和半熟素材——比喻、事实、故事、图表。到正式准备讲演时，所有素材触手可及，只需串联即可。
- **结果**："我现在可以自如掌控轻重缓急——可以优先处理工作、家庭以及个人感情中的要紧事，然后只须关注此时此刻手头正在处理的事项即可。"

### 案例 3：帕特里克牧师的悼念仪式
- **问题**：科罗拉多州某教堂的牧师帕特里克需要策划悼念仪式——通过追述逝者一生体现生命的荣耀和意义，以前这是一项繁重的工程。
- **方法论的使用**：使用录音转文字记录与逝者家属的访谈；将所有素材（谈话记录、讣告、照片）集中保存在项目文件夹；每次访谈后花15分钟筛选最打动人心的话。
- **结果**：当他将所有素材归整到一起时，仪式便已完成了80%。"只把最精华的素材喂给大脑，对我来说是一种自由。"

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 每次写年终总结、季度报告时都从零开始，痛苦不堪
2. 跳槽/换岗时觉得自己"几年下来没什么积累"
3. 做重复性工作时（如每次活动策划都重新写方案）感到效率极低
4. 面对一个大项目不知如何下手，觉得工程量太庞大了
5. 创意工作（写作、设计、编程）中频繁遇到"空白页恐惧"

### 语言信号 (用户的话里出现这些就应激活)

- "每次做PPT/报告都从零开始"
- "以前做过但找不到了"
- "怎么复用以前的工作"
- "重复造轮子"
- "觉得做了很多但没有积累"
- "从零开始太难了"
- "总是无法完成大项目"

### 与相邻 skill 的区分

- 与 `para-system` 的区别：PARA管"放在哪"，半熟素材管"怎么用"。半熟素材更偏重工作产出的资产化思维。
- 与 `progressive-summarization` 的区别：渐进式归纳法产出的高亮笔记是半熟素材最常见的原材料之一。
- 与 `creative-workflow` 的区别：半熟素材是"素材的思维"，创造性工作流是"流程的思维"。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **识别用户已经拥有但未意识到的半熟素材**
   - 让用户回想最近完成的一个项目（工作或个人），列出该过程中产生的所有"中间产物"
   - 引导识别：你写了什么文档？做了什么表？画了什么图？整理了哪些数据？收到了哪些反馈？
   - 完成标准：列出至少5个潜在半熟素材

2. **将素材分类并保存**
   - 帮助用户将识别出的素材按五种类型（干货笔记/落选素材/过程文件/最终成果/他山之石）分类
   - 引导用户将这些素材从"已完成项目"文件夹中提取出来，放入对应的PARA项目/领域/资源文件夹
   - 完成标准：至少3个素材完成了分类和重新存放

3. **实践"永不从零开始"原则**
   - 让用户找出一个即将启动的新项目
   - 先搜索第二大脑中所有可能相关的半熟素材，拖入新项目文件夹
   - 在素材基础上搭建初始框架，而非面对空白页面
   - 完成标准：新项目有了一个基于已有素材的初始框架，而非空白文档

4. **培养"项目完成≠知识丢弃"习惯**
   - 介绍项目完成清单（来自knowledge-lifecycle skill）中的第3步：重温半熟素材并移至其他文件夹
   - 完成标准：用户承诺在下一个项目完成时花5-10分钟执行知识回收

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 一次性的、确定永不复用的简单任务（如回复一封简单邮件）——不必为一切创建模板
- 需要严格保密的内容（如商业机密、个人隐私数据）——这些不适合作为可复用的开放素材
- 仍在快速迭代中的创作原型——过早资产化可能限制创意灵活性

### 作者在书中警告的失败模式

- **"还没准备好"的迷思**：认为只有收集到所有可能的半熟素材才能开始创作——这是拖延的变体。实际上，你手头的素材通常已经足够启动。
- **消费主义信息观**：只积累不创造——半熟素材的目的是被使用，而非被收藏。

### 作者的盲点 / 时代局限

- 偏向独立创作者的工作模式，在严格的企业合规/保密环境中的适用性需要调整
- 素材的版权和出处管理问题（尤其是"他山之石"类型）讨论较浅

### 容易混淆的邻近方法论

- 模板思维（Templates）：模板是半熟素材的一种（最终成果类型），但半熟素材的范围远大于模板——它还包括落选素材、过程文件、他山之石等。
- 知识库（Knowledge Base）：知识库是静态的"信息集合"，半熟素材是动态的"创作单元"——一个是为了查阅，一个是为了组装。
