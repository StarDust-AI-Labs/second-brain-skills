# 发散与聚合模式 (Divergence & Convergence)

## R — 原文 (Reading)

> "如果对创作活动的过程仔细观察，便会发现，它总是遵循着同一种简单的模式，即在'发散'与'聚合'之间往复交替。创作过程的起点是'发散'行为，即为各种可能性打开空间，并考虑尽可能多的选项……到了'聚合'阶段，我们不得不淘汰各种选择，仔细权衡并最终决定什么才是真正重要的。"
>
> —— 蒂亚戈·福特，《打造第二大脑》第八章

> "'信管法则'的前两个步骤——抓取和组织，构成了'发散'过程。这一阶段的主题是研究、探索和萌发创意。后面两个步骤——提炼和表达，则构成了'聚合'过程。这时，我们停止吸纳新思想，并利用收集完毕的知识模块创建新事物。"
>
> —— 《打造第二大脑》第八章

## I — 方法论骨架 (Interpretation)

发散与聚合是所有创造性活动底层通用的双模式节律。它描述了人类创作活动的本质：先打开选项空间，后关闭选项空间。

**发散模式**（打开）：

- 心态：开放的、好奇的、不评判的
- 行为：收集素材、头脑风暴、浏览资料、探索新方向、与各种人聊天
- 在CODE中对应：抓取 + 组织
- 目标：尽可能多地积累"可能性"——你无法预测哪颗种子会发芽

**聚合模式**（关闭）：

- 心态：果断的、取舍的、务实的
- 行为：筛选材料、删减内容、排列优先级、决定方向、完成交付
- 在CODE中对应：提炼 + 表达
- 目标：从众多可能性中选出那个唯一，并把它变成真实的产出

**为什么大多数人卡在发散模式**：
发散给人探索的快感和"我在进步"的幻觉。聚合则要求做减法、忍痛割爱——砍掉一个好想法比想出一个新想法痛苦得多。你越有创造力、好奇心越强，切换越难。

**切换信号**：

- 当素材开始重复（"怎么又是类似的观点"）→ 你该聚合了
- 当你开始焦虑"做不完" → 你该聚合了
- 当你的资料库持续增长但没有任何可交付物 → 你已经发散太久了
- 当你发现自己在做决定时还在搜索新信息 → 你在用发散逃避决策

## A1 — 书中的应用 (Past Application)

### 案例 1：车库工作室改造项目中的发散与聚合

- **问题**：作者计划将车库改造为家庭工作室，项目启动时只知大概方向，不知具体该做什么。
- **方法论的使用**：发散阶段：创建项目文件夹，花15分钟做头脑风暴大纲（思想群岛）；随后数周收集灵感——Pinterest照片、朋友建议、YouTube教程、邻居推荐的承包商名单。聚合阶段：发现想法已经扩展到敲墙凿顶、重构后院等不切实际的程度，于是使用压缩范围策略——设定预算和时限，砍掉"梦想清单"中的大部分，只保留核心功能。
- **结果**：项目从空想变成了实际可行、可交付的计划。

### 案例 2：毕加索的《公牛》——去粗取精的极致

- **问题**：毕加索1945年创作《公牛》系列版画，探索如何表现公牛的"本质形态"。
- **方法论的使用**：前几幅画是发散——加入大量细节：饱满的牛角、立体的尾巴、厚实的皮肉质感。从第4幅开始聚合——用锐利白线勾勒肌肉、简化头部细节、舍弃装饰性笔画。到最后一幅，只剩寥寥数笔，却将公牛的神韵勾勒得淋漓尽致。
- **结果**："结果之'简'掩盖了过程之'繁'"——毕加索不是一开始就用线条作画，而是经过了完整的发散→逐步聚合的创作过程。

### 案例 3：纪录片导演的40:1素材比

- **问题**：著名纪录片导演肯·伯恩斯在制作《美国内战》《爵士乐》等获奖影片时，积累了大量原始镜头。
- **方法论的使用**：发散阶段拍摄数百小时的素材；聚合阶段采用激进的筛选策略——平均每40到50小时的视频素材中，仅有1小时内容被剪入最终成片。
- **结果**：多次获奖。这证明了专业创作者的聚合比普通人想象的要大胆得多——你不需要保留一切。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 准备写一篇文章/报告/方案，收集了大量素材但迟迟不动笔
2. 项目计划阶段花费时间远超预期，总感觉"还需要更多研究"
3. 做决策时不断搜索新信息，却越来越难下决定
4. 创意人的"项目墓地"——开了很多头但没一样完成
5. 完美主义导致交付拖延——害怕提交"不够好"的作品

### 语言信号 (用户的话里出现这些就应激活)

- "素材收集够了但就是写不出来"
- "总觉得还差一点"
- "不知道怎么收尾"
- "一直在看资料但做不出决定"
- "开了一堆项目没一个完成的"
- "不会做减法"
- "完美主义让我动不了"

### 与相邻 skill 的区分

- 与 `creative-workflow` 的区别：发散与聚合是底层模式解释（"为什么你会卡住"），创造性工作流是具体操作技术（"怎么办"）
- 与 `second-brain-code` 的区别：发散与聚合解释了CODE的内部节奏，CODE是外部步骤
- 与 `capture-criteria` 的区别：抓取标准用于发散阶段（"什么值得收集"），发散聚合帮你判断"何时该停止收集"

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **诊断用户当前处于哪种模式**
   - 询问用户最近一周的活动：更多是在收集新信息/探索新方向（发散），还是在筛选/删减/交付（聚合）？
   - 询问：你手头有没有一个"一直在进行但从未完成"的项目？
   - 完成标准：明确用户卡在哪个模式、卡了多久
2. **如果用户卡在发散模式，引导切换至聚合**
   - "发散关闭仪式"：宣布"发散阶段到此结束"，关闭所有新信息入口——不再搜索、不再打开新标签、不再看相关文章
   - 设定一个具体的最小交付目标（参考creative-workflow的压缩范围）
   - 在已有素材中选择最好的5-7个（而非继续收集新素材）
   - 完成标准：用户执行了至少一个"关闭新信息入口"的动作
3. **如果用户卡在聚合却不满意结果，引导重新发散**
   - 有时聚合太早也会出问题——你还没收集够好素材就开始做决定
   - 判断标准：如果你能轻松列出3个以上完全不同的方案，说明发散还不够
   - 给发散一个时间框（如"再花30分钟收集3个新灵感来源"），然后再次切换
   - 完成标准：设定了一个明确的一轮发散时间限制
4. **建立模式切换的日常节奏**
   - 推荐：每天上午发散（阅读、浏览、收集），下午聚合（写作、决策、交付）
   - 或：一周的前半周发散（研究），后半周聚合（产出）
   - 关键：不要同时做两种模式——它们调用不同的认知资源
   - 完成标准：用户设计了一个适合自己节奏的发散/聚合时间划分

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 纯执行类任务（如数据录入、按流程审批）——这些工作没有"可能性空间"，不需要发散
- 紧急危机处理——发散需要心理安全感，危机时刻应直接进入聚合决策
- 用户只是偶尔拖延而非系统性"无法交付"——发散聚合是一个创作模式问题，不是拖延症治疗方案

### 作者在书中警告的失败模式

- **无法从发散切换到聚合**：好奇心越强、兴趣越多样、对完美标准越高的人，切换越困难。"所有看上去很美的推动因素实际上都属于发散行为，甚至可能会对项目的如期完成产生阻碍作用。"
- **近期偏差**：在聚合决策时，被最近接触的信息（而非长期以来最优质的信息）所绑架。应使用第二大脑的历史积累对抗这种倾向。

### 作者的盲点 / 时代局限

- 发散/聚合模型源自设计思维理论，作者的主要贡献在于将其与CODE叠加和应用到个人知识管理，而非原创该模型
- 对于必须在高压环境下持续并行多项目的实际工作者，是否有足够的心理空间进行模式切换存在疑问

### 容易混淆的邻近方法论

- 双钻模型 (Double Diamond)：设计思维中的"发现-定义-开发-交付"四阶段，本质上是两次"发散→聚合"的串联。发散与聚合更底层、更通用。
- 深度工作 vs 浅层工作：Cal Newport的二分法关注的是注意力深度，发散聚合关注的是思维模式的开放性。两者交叉但不重叠。
