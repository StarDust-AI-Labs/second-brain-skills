  title: 信管法则 (CODE)
  type: framework
  source_chapter: 第三章
  source_quote: |
    "为了帮助大家构建起属于自己的第二大脑，我发明了一个简单而直观的四步法，也即所谓的'信管法则（CODE）'——抓取（Capture）、组织（Organize）、提炼（Distill）、表达（Express）。"
  summary: |
    CODE法则是全书最顶层的操作框架，将信息管理分为四个顺序步骤。前两步（抓取+组织）属于"发散"模式，后两步（提炼+表达）属于"聚合"模式。这四个步骤既是入门方法，也是长期策略，适用于任何职业、角色和信息类型。用户可以用它作为导航图，在信息海洋中遵循"收集→整理→浓缩→输出"的路径将碎片知识转化为创造性成果。
  tags: [信息管理, 工作流, 创造力, 生产力]

- id: f02
  title: PARA系统
  type: framework
  source_chapter: 第五章
  source_quote: |
    "我将这种组织方法命名为PARA系统，它很好地概括了生活中常见的四大信息类型，也即项目（Project）、领域（Area）、资源（Resource）以及存档（Archive）。"
  summary: |
    PARA系统是按"可行动性"而非"主题"组织数字信息的分类框架。Project是正在从事的短期有终点的任务；Area是需要长期负责但无终点的生活/工作领域；Resource是兴趣话题的参考材料；Archive是已完成或搁置的休眠信息。四类的优先级按"行动力"从高到低排列，信息可在四类之间灵活流动。该系统的核心洞见是：以行动为导向比以主题为导向更能驱动知识的实际使用。
  tags: [信息组织, 分类系统, 生产力, 知识管理]

- id: f03
  title: 渐进式归纳法 (Progressive Summarization)
  type: framework
  source_chapter: 第六章
  source_quote: |
    "'渐进式归纳法'是我大力提倡的一套笔记提炼技巧……这种技巧说来也简单：先将某条笔记的要点标记出来，然后再标记出要点的要点，以此类推，便能层层提炼出笔记的精华所在。由于每一层级的提炼都会采用不同的格式，所以很容易加以区分。"
  summary: |
    一种分层的笔记提炼技术，包含四个层级：第1层是原始摘录（"土壤"），第2层是黑色加粗的要点（"原油"），第3层是黄色高亮的精华（"黄金"），第4层是用自己语言写的纲要（"宝石"）。每个层级不应超过上一层的10%-20%。核心逻辑是"断舍离"而非"滴水不漏"——为未来的自己节省重复阅读的时间，根据当时的可用时间灵活选择深入哪一层级。
  tags: [笔记方法, 信息提炼, 学习技巧, 知识可见性]

  type: framework
  source_chapter: 第七章
  source_quote: |
    "这里所说的系统即指我们的第二大脑。至于其中收录的所有零碎信息的集合，我愿称之为'半熟素材'，也即构建最终作品所必需的、个别的、具体的创作元素。"
  summary: |

  tags: [创作方法, 知识资产, 生产力, 模块化思维]

- id: f05
  title: 发散与聚合模式切换 (Divergence & Convergence)
  type: framework
  source_chapter: 第八章
  source_quote: |
    "如果对创作活动的过程仔细观察，便会发现，它总是遵循着同一种简单的模式，即在'发散'与'聚合'之间往复交替。创作过程的起点是'发散'行为，即为各种可能性打开空间，并考虑尽可能多的选项……到了'聚合'阶段，我们不得不淘汰各种选择，仔细权衡并最终决定什么才是真正重要的。"
  summary: |
    描述创作活动底层规律的双模式模型。发散是打开可能性、收集素材、探索新路径的扩展阶段；聚合是筛选淘汰、聚焦交付、完成最终方案的收缩阶段。CODE的前两步（抓取+组织）对应发散，后两步（提炼+表达）对应聚合。两者并非线性顺序，而是循环往复迭代进行。最重要的挑战是意识到自己当前处于哪种模式，并有勇气从发散切换到聚合。
  tags: [创作过程, 思维模式, 决策框架, 创新方法论]

  type: framework
  source_chapter: 第四章
  source_quote: |
    "你需要在脑海中整理出一打最感兴趣的问题……每次当你听说或者学习到某些新发现或新技术时，就用这十二个问题进行测试，看看能否对其中的一项或几项有所帮助，那么每隔一段时间便会获得一些突破。"

  tags: [信息筛选, 好奇心管理, 费曼方法, 长期思维]

  title: 四项信息抓取标准 (Four Capture Criteria)
  type: framework
  source_chapter: 第四章
  source_quote: |
    "我在此建议采用以下四大标准帮助你判断何种信息是适宜记录的。1. 是否具有启发性；2. 是否具有实用性；3. 是否具有个性；4. 是否具有新奇性。"
  summary: |
    判断信息是否值得保存的四维度筛选框架。启发性：能激发灵感、提供动力；实用性：未来可能派上用场的知识工具；个性：反映个人观点、反思、记忆；新奇性：挑战既有认知、带来意外发现。四条标准各自独立但互补，共同防止"数字囤积"——只收集而不消化。最终极标准是"共鸣"：是否触动内心直觉。
  tags: [信息筛选, 决策标准, 知识管理, 内容策展]

  type: framework
  source_chapter: 第四章
  source_quote: |
    "你在日常与各种内容打交道时，请留意自己的内心有没有被某条信息打动或是惊艳到。这种特别的共鸣感——如同来自灵魂深处的回响，便是直觉在向你发出强烈提醒：一定要把它记录下来！"
  summary: |
    信息抓取的终极标准。作者认为，使用冗长清单进行理性分析会消耗过多脑力，而直觉驱动的"共鸣感"是一种更高效、更准确的筛选机制。当信息让你心跳加速、眼睛瞪大、世界静止时，就应记录。神经科学证据表明，情感引导而非扰乱理性思维，潜意识可以自行学习并做出反应，早于理性判断。长期练习可以强化直觉能力，使之在决策、目标设定等方面发挥更大作用。

- id: f09
  title: 思想群岛 (Archipelago of Ideas)
  type: framework
  source_chapter: 第八章
  source_quote: |
    "在我面前摆放着的不是一张张空空如也的白纸，而是一整套资料翔实的文档……现在，每当我开启一个新篇章时，各种素材便如同星罗棋布的群岛一样围绕着我，而我只须在各种'岛屿'之间搭建好桥梁。如此一来，创作活动也变得不再那么令人生畏。"
  summary: |
    源自作家史蒂文·约翰逊的创意大纲构建技术。在发散阶段收集足够多的素材（笔记、观点、案例、引用），将它们排列在一起形成"群岛"，然后在聚合阶段在这些"岛屿"之间搭建逻辑桥梁。与传统大纲相比，数字版思想群岛更具延展性（可自由增删重组）、可链接至详细内容、支持多媒体，且易于搜索。核心价值在于分离"选择想法"（发散）和"逻辑排序"（聚合）两个认知活动。
  tags: [创意大纲, 写作方法, 认知分工, 项目管理]

- id: f10
  title: 海明威之桥 (Hemingway Bridge)
  type: framework
  source_chapter: 第八章
  source_quote: |
    "海明威……一旦他明确了后续章节的写作思路，便会立刻停止当前章节的写作。换言之，他会在新思路诞生的第一时间切换至该部分的写作，而非等到当前的文思耗尽才转至下一阶段。他用今天的思路和精力为明天的写作注入动力，这种策略使得他每次开始写作活动时都能做到有的放矢。"
  summary: |
    跨工作阶段的意图传递技术。在每个工作阶段结束前，有意留下"未完之事"：写下下一步的想法、当前状态（最大挑战、未决事项）、易被遗忘的细节、下一阶段的工作目标。下次重启时不必从冷启动开始，而是有明确的落脚点和行动基础。还可将草稿分享给他人获取反馈，增加重启时的素材。核心机制是利用"蔡格尼克效应"（对未完成任务的记忆优势）保持思维连续性。
  tags: [工作流, 写作技巧, 认知连续性, 时间管理]

  title: 压缩范围 (Scope Compression)
  type: framework
  source_chapter: 第八章
  source_quote: |
    "我们完全可以将项目的范围压缩在可控的规模之内，事实上这也是确保成功的必由之路。我们不应坐等所有的准备完全就绪才展开行动……学会放弃、削减，或是推迟那些不太重要的部分，我们便可坦然应对紧迫的时限，不断取得新的突破。"
  summary: |
    源自软件开发的项��管理策略。面对复杂项目时，不是无限推迟发布日期，而是主动对功能进行压缩：砍掉难度最大、成本最高、不确定性最强的部分，保留核心功能先行交付，其余留待后续迭代。关键配套是第二大脑——被删减的材料不会丢失，而是被妥善保存以备未来项目复用（可能成为激活另一个项目的关键）。核心心态是从"完美交付"转向"先交付最小可行版本，再迭代完善"。

- id: f16
  title: 四种检索方法 (Four Retrieval Methods)
  type: framework
  source_chapter: 第七章
  source_quote: |
    "这种天然的不可预测性，意味着在搜索笔记信息时，不能仅仅依赖某个单一的检索系统，而应搭配使用以下四种检索方法……1. 搜索；2. 浏览；3. 标签；4. 随缘。"
  summary: |
    从第二大脑中调取信息的多策略组合框架。搜索：最高效直接，适合已知目标信息；浏览：通过文件夹层级逐步深入，利用人类更擅长的空间思维能力；标签：打破文件夹的物理隔离，建立跨分类关联，适合临时性项目；随缘：通过视野拓展、视觉浏览和社交分享创造"机缘巧合"，让不同领域的想法意外碰撞。四种方法各有适用场景，搭配使用比单一方法效果更好。
  tags: [信息检索, 搜索策略, 知识发现, 机缘巧合]

- id: f18
  title: 知识复利思维 (Knowledge Compounding)
  type: framework
  source_chapter: 第九章
  source_quote: |
    "资产可以带来回报，而回报又可作为再投资创造其他价值。知识同样也可以随着时间推移而产生收益和复利，从而成为一种高回报的资产。就像每个月都在股市里投入一小笔资金一样，你在知识领域投入的每一分精力，都让你变得更加博学和聪慧。"
  summary: |

  tags: [复利思维, 知识投资, 长期主义, 系统思维]

    "学术界将这种现象命名为'大教堂效应'。研究发现，环境会对个体思维模式产生重大影响。当人们身处开阔空间……思维格局会被明显打开；而当人们处于相对逼仄的空间内时，思考的内容则会变得更加具体。"
  summary: |

- id: f20
  title: 从稀缺到充裕的思维转变 (Scarcity to Abundance Mindset)
  type: framework
  source_chapter: 第十章
  source_quote: |
    "我们的思维模式将从基于稀缺性的角度看问题（'稀缺性思维'）转向基于充裕性的角度看问题（'充裕性思维'）。……稀缺性思维认为信息是稀缺资源，要尽可能多地收集……充裕性思维认为，不必吸收和理解全部的信息，只须从中汲取一些智慧的灵感。"
  summary: |
    描述信息管理底层心态转变的二极模型。稀缺性思维表现为：害怕遗漏、无差别囤积信息、认为价值来自信息占有量。充裕性思维表现为：相信有价值的信息无处不在、只需汲取灵感而不必穷尽、敢于舍弃不必要的信息。转变的关键在于认识到：知识的价值在于分享和使用，而非囤积；信息输入会源源不断（不必恐慌性收集）；放弃低价值信息不会造成损失。这一转变是从"消费者"到"创作者"身份转换的心理基础。
  tags: [心态转变, 信息焦虑, 充裕心态, 极简主义]

- id: f21
  title: 好莱坞模式 (Hollywood Model)
  type: framework
  source_chapter: 第五章
  source_quote: |
    "这种项目中心化的方式也渗透到其他类型的脑力工作领域，并形成一股名为'好莱坞模式'（Hollywood model，得名于电影制片领域）的潮流。确定好项目、组建好队伍、通力合作直至完成工作，然后江湖再见。"
  summary: |
    以项目为中心的现代工作组织模式。源于电影制片的运作方式：围绕具体项目组建临时团队，项目完成后团队解散。与之对应的是PARA系统中将"项目"作为最基本组织单位的方法。核心逻辑是：工作应围绕有明确起点和终点、有具体交付物的"项目"来组织，而非永无止境的"岗位职责"。这使工作更具辨识度、更容易衡量进展，也更便于在不同项目间转移和复用知识资产。
  tags: [工作模式, 项目化管理, 组织设计, 敏捷思维]
