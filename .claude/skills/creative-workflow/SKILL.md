# 创造性工作流（思想群岛 + 海明威之桥 + 压缩范围）

## R — 原文 (Reading)

> "在我面前摆放着的不是一张张空空如也的白纸，而是一整套资料翔实的文档，它们来自各种信件、原始文献、学术论文以及我自己的笔记。这真的是对抗拖延症的绝佳技巧……各种素材便如同星罗棋布的群岛一样围绕着我，而我只须在各种'岛屿'之间搭建好桥梁。"
>
> —— 史蒂文·约翰逊，转引自《打造第二大脑》第八章

> "海明威……一旦他明确了后续章节的写作思路，便会立刻停止当前章节的写作。他用今天的思路和精力为明天的写作注入动力，这种策略使得他每次开始写作活动时都能做到有的放矢。"
>
> —— 《打造第二大脑》第八章

> "我们不应坐等所有的准备完全就绪才展开行动，正如不应坐等所有的交通灯同时变绿才发动汽车一样。学会放弃、削减，或是推迟那些不太重要的部分，我们便可坦然应对紧迫的时限。"
>
> —— 《打造第二大脑》第八章

## I — 方法论骨架 (Interpretation)

创造性工作流由三个互补的策略组成，分别解决创作过程的前、中、后三个阶段的典型障碍：

### 策略1：思想群岛（解决"空白页恐惧"——怎么开始？）

在发散阶段，从第二大脑中收集已有的相关笔记和半熟素材，把它们排列在一起，形成一座座"岛屿"。你的任务不再是"从零开始创作"，而是在岛屿之间搭建逻辑桥梁。

**与传统大纲的核心区别**：传统大纲要求同时完成"选择想法"和"逻辑排序"两个认知活动——这几乎不可能高效完成。思想群岛将它们分离为两个独立步骤：先收集岛屿（发散），再搭建桥梁（聚合）。

**数字版优势**：可自由添加/删除/重组要点、可链接到详细参考笔记、支持多媒体嵌入、可在各设备间同步和编辑。

### 策略2：海明威之桥（解决"重启困难"——下次怎么接上？）

每次工作阶段结束时，不要在精力完全耗尽时才停下。在你明确知道"下一步要做什么"的时候，主动停下来，并写下：
- 下一步的想法和思路
- 当前的状态（最大挑战、未决事项）
- 易被遗忘的细节（角色性格、场景伏笔、设计巧思）
- 下一阶段的工作目标

下次（无论是一天后还是几个月后）重启时，你不是从零冷启动，而是有明确的入口和动力。你还可以在暂停期间将草稿分享给他人获取反馈，下次重启时便有了更多素材。

### 策略3：压缩范围（解决"交付恐惧"——怎么收尾？）

当项目变得过于庞大和复杂时，不要无限延期。主动砍掉最困难、最昂贵、最不确定的部分，只保留核心功能先行交付，其余留待后续迭代。关键配套：被砍掉的材料存入第二大脑——它们不会消失，可能在未来的某个项目中华丽归来。

**三者配合使用**：思想群岛帮你起步 → 海明威之桥帮你保持连续性 → 压缩范围帮你到达终点。

## A1 — 书中的应用 (Past Application)

### 案例 1：作者撰写本书时的思想群岛
- **问题**：作者需要写一篇关于"札记本"的深度文章。
- **方法论的使用**：他从第二大脑中提取了所有关于札记本的笔记——约翰·洛克的札记本方法、画家查克·克洛斯的栅格创作法、自己以前写过的相关文章片段——排列成一个要点式大纲（思想群岛），并设置了链接指向每条资料的详细笔记。
- **结果**：呈现在他面前的不是空白页面，而是一整套资料翔实的素材网。只需在各要点之间搭建逻辑桥梁即可成文。

### 案例 2：海明威的写作策略
- **问题**：欧内斯特·海明威每天需要面对长篇小说的写作——如何保证每天都能迅速进入状态？
- **方法论的使用**：一旦明确了下一章节的写作思路，他就立刻停止当前写作——即使文思正酣。他会把"下一步"留在脑海中作为明天的入口。
- **结果**：海明威成为20世纪最具影响力的作家之一，1954年获诺贝尔文学奖。他的这种策略被称为"海明威之桥"。

### 案例 3：作者的车库改造项目整合实战
- **问题**：作者计划将车库改造为家庭工作室，项目规模迅速膨胀——从简单的改造到敲墙凿顶、重构后院、搭建超高速网络。
- **方法论的使用**：
  - 思想群岛：创建项目文件夹，花15分钟列出大纲（简介、成本、设想、阶段、美学、设备等），附上参考照片、朋友建议、YouTube教程等链接
  - 海明威之桥：每次只能花碎片时间推进时，在笔记中记录学习进度和下一步，确保下次能无缝衔接
  - 压缩范围：与妻子一起设定预算和时限，砍掉最不切实际的想法，将范围压缩到可控水平
- **结果**：项目从空想落地为实际可行、可交付的计划。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 面对空白文档/PPT/画布，盯着看了半天一个字也写不出来（用思想群岛）
2. 业余项目（写作/编程/设计）每次都虎头蛇尾——开头兴致勃勃，过几天就再也不想打开（用海明威之桥）
3. 接了一个大项目但不知从何入手，觉得工程量太庞大（组合使用三种策略）
4. 完美主义导致永远在"打磨"而不交付——害怕作品被批评（用压缩范围）
5. 碎片时间无法利用——总觉得需要整块时间才能做创意工作（用海明威之桥 + 半熟素材）

### 语言信号 (用户的话里出现这些就应激活)

- "不知道怎么写开头"
- "空白页恐惧"
- "项目太大不知从何下手"
- "总是做一半就不想做了"
- "每次重新打开项目都忘了上次做到哪"
- "怎么把大项目拆小"
- "完美主义让我交不出东西"
- "需要一个创作流程"
- "虎头蛇尾"

### 与相邻 skill 的区分

- 与 `diverge-converge` 的区别：发散聚合是底层理论模型（解释"为什么"），创造性工作流是实操工具集（解决"怎么做"）
- 与 `intermediate-packets` 的区别：半熟素材是创作原材料，创造性工作流是如何使用这些原材料的方法
- 与 `knowledge-lifecycle` 的区别：创造性工作流侧重"创作过程中的技巧"，知识生命周期侧重"项目的启动和收尾标准化"

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

<HARD-GATE id="blocker-diagnosed-before-workflow">
未诊断创作卡点之前，不得直接给大纲、草稿或任务清单。

必须先归类为:
1. 开始困难 → 思想群岛
2. 持续困难 → 海明威之桥
3. 收尾困难 → 压缩范围

禁止的行为:
- 默认用户只是需要大纲
- 同时套用三个策略但不说明主攻方向
- 用通用写作建议替代创作流程
</HARD-GATE>

<HARD-GATE id="next-artifact-before-finish">
结束前必须产出一个可继续推进的中间产物。

至少满足一项:
1. 思想群岛素材清单
2. 海明威之桥下一步记录
3. 最小可交付范围

禁止的行为:
- 只解释方法论不落到产物
- 只鼓励用户继续收集素材
- 没有下一步动作就结束
</HARD-GATE>

1. **诊断用户的创作障碍**
   - 询问：你现在卡在创作的哪个阶段？是不知道如何开始、中途无法持续、还是无法收尾？
   - 完成标准：确定推荐哪个策略作为主攻方向

2. **若卡在"开始"→使用思想群岛**
   - 搜索第二大脑中与该主题相关的所有现有笔记和半熟素材
   - 将它们集中到一个空白笔记中，排列成要点列表（不需要逻辑顺序，先堆上去）
   - 将相关的参考笔记链接附在每个要点后面
   - 然后再做逻辑排序和桥梁搭建
   - 完成标准：用户面前不再是空白页面，而是一个至少有5个要点的素材大纲

3. **若卡在"持续"→搭建海明威之桥**
   - 在当前工作阶段结束前（或在下次重启前回顾上次的状态），写下：
     - 下一步要做什么（具体到可执行的动作）
     - 当前最大的未决问题
     - 任何容易被遗忘的细节
   - 如果项目已中断，引导用户先读这些"桥"再开始工作
   - 完成标准：建立了一个明确的"下次入口"，下次打开项目时30秒内能知道该做什么

4. **若卡在"收尾"→使用压缩范围**
   - 列出项目的所有功能/内容/子任务
   - 用"必要→重要→锦上添花"三档分类
   - 将"重要"和"锦上添花"的条目移入第二大脑保存（而非删除），设定为"后续迭代"
   - 只交付"必要"部分
   - 完成标准：用户已确定了最小可交付范围

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 简单重复性任务（填表、签到、常规审批）——这些不需要创意工作流
- 用户已经接受了严格的敏捷/Scrum方法论且有Scrum Master支持——压缩范围的理念已内嵌其中
- 用户对创作过程没有持续性问题——只是在某一天状态不好，这是正常波动

### 作者在书中警告的失败模式

- **FOMO（错失恐惧症）**：在思想群岛阶段，总觉得还应该再收集更多素材才能开始。提醒：你是在做雏形，不是最终作品。
- **"时机不成熟"拖延**：把压缩范围当作"不用交付"的借口。压缩范围的目的是交付更小的版本，而非无限期推迟。
- **海明威之桥写成流水账**：不是记录"今天做了什么"，而是记录"下次该做什么+当前状态+易忘细节"。

### 作者的盲点 / 时代局限

- 偏向独立创作者的工作模式，团队协作中的集体创意流程（如小组头脑风暴的过渡机制）讨论不足
- 海明威的写作策略适合长篇创作，对于社交媒体时代的微内容创作（一条推文、一张图）的适用性需要重新评估

### 容易混淆的邻近方法论

- 思维导图 (Mind Mapping)：思维导图是发散工具，思想群岛同时服务于发散和聚合——它既是素材的集合，也是逻辑的框架。
- Pomodoro番茄工作法：番茄工作法管理的是"注意力持续时间"，海明威之桥管理的是"跨工作阶段的意图连续性"。

## 合理化防御（Prebuttal）

Agent 在压力下会为跳过创意工作流步骤找到合理理由。以下是预先反驳：

| Agent 的借口 | 预先反驳 |
|-------------|---------|
| "用户想法很清晰了，直接写就行" | 清晰的想法也需要"思想群岛"做结构化。跳过素材收集 = 从零开始的恐惧会卷土重来 |
| "这个项目很小，不需要完整流程" | 小项目也会卡住。海明威之桥对任何规模的创作都有用——知道下一步从哪开始，就是不卡死的保证 |
| "用户着急要结果，跳过中间步骤更快" | 跳过步骤现在看起来更快，但半途卡住的花费远超前置投入。创作工作流的价值就是防止卡住 |
| "素材够多了，不需要再收集" | 素材够了≠素材结构化了。思想群岛的核心不是收集更多，是把已有的连接成可导航的群岛 |
| "压缩范围会让内容太少" | 压缩范围是聚焦，不是删减。缩小范围 → 提高深度 → 作品质量提升。小而深 > 大而浅 |
| "这是突发灵感，不用走完整流程" | 灵感最需要结构化。没有流程的灵感 = 写了一半就放弃的草稿 |
