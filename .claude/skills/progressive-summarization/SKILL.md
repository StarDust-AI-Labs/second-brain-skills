# 渐进式归纳法 (Progressive Summarization)

## R — 原文 (Reading)

> "'渐进式归纳法'是我大力提倡的一套笔记提炼技巧……这种技巧说来也简单：先将某条笔记的要点标记出来，然后再标记出要点的要点，以此类推，便能层层提炼出笔记的精华所在。由于每一层级的提炼都会采用不同的格式，所以很容易加以区分。"
>
> —— 蒂亚戈·福特，《打造第二大脑》第六章

> "关于画重点的一个有效的经验法则就是，每一层级的重点不应超过上一层级内容的10%～20%。如果什么都想要，最后就什么都得不到。"
>
> —— 《打造第二大脑》第六章

---

## I — 方法论骨架 (Interpretation)

渐进式归纳法是一套**按需深入**的笔记提炼系统——你不必一次性把笔记处理到完美，而是随着每次"重逢"逐步加深提炼。

四个层级，由粗到精：

**第1层 — 原始摘录（"土壤"）**：你从原文中截取的最精华段落。这是地基，只保留最有价值的部分，通常不超过原文的10%。例如从一篇5000字的文章中只抓500字。

**第2层 — 加粗要点（"原油"）**：对第1层的精选内容，将核心关键词、关键句加粗。一个有效的标准：只浏览加粗部分就应该能理解整篇笔记的大意。加粗量不超过第1层的10-20%。

**第3层 — 高亮精华（"黄金"）**：在加粗内容的基础上，对最令人惊叹、最具可操作性的部分做高亮标记。这是"百里挑一"——整篇笔记中配得上高亮的不过寥寥数句。高亮量不超过加粗内容的10-20%。

**第4层 — 纲要总结（"宝石"）**：用自己的话在笔记顶部写一份项目符号式的纲要。这是终极提炼——不需要看原文就能迅速回忆起全部要点。只需对极少数极其重要、反复使用的笔记做此处理。

**关键原则**：
- **时机原则**：只在准备创作（写文章、做演示、准备会议）时才深度提炼。不要漫无目的地"清理"所有笔记。
- **接触即改善**：每次因任何原因打开一条笔记时，至少做一点改进——加一个标题、标一处重点、写一句注释。
- **渐进原则**：四个层级不是一次完成的，而是跨数周、数月甚至数年的多次"偶遇"中层层叠加。

## A1 — 书中的应用 (Past Application)

### 案例 1：经济学笔记的跨时空重逢
- **问题**：作者多次因为同一个晦涩的经济学术语"鲍莫尔成本病"而反复查阅维基百科，每次都要重新读一遍。
- **方法论的使用**：在第一次查阅时，将维基百科条目的精华部分摘录为笔记（第1层）。数月后再次遇到这个术语时，花了些工夫将关键词加粗（第2层），将核心概念高亮（第3层）。
- **结果**：后来参加一场座谈时，演讲者提到这一术语，作者仅用10秒钟搜索并调取笔记，就能胸有成竹地做出回应，"俨然一副资深学者的气派"。

### 案例 2：提前两年的招聘准备
- **问题**：作者读到一篇关于谷歌"结构化面试"的文章时，还是自由职业者，对招聘没有紧迫需求。
- **方法论的使用**：将文章核心摘录保存在"招聘"资源文件夹中（第1层）。两年后，当他需要招聘第一名员工时，将该文件夹从资源转为项目，花半小时温习笔记并加粗和高亮最相关的内容（第2-3层）。
- **结果**：从谷歌的最佳实践中获得启发，成为他开展招聘活动的出发点和落脚点，大大减轻了第一次做管理者的焦虑。

### 案例 3：播客笔记的延时提炼
- **问题**：作者在开车旅行途中收听到一位创业者的精彩播客访谈，无法及时记录。
- **方法论的使用**：抵达目的地后，把脑海中印象最深的要点记录下来（这说明最精华的内容可以在脑海中存活1-2小时——这是一种自然的"去粗取精"）。几周后需要上线新课程时，重温笔记、加粗关键定价策略、高亮适用部分。
- **结果**：直接借鉴了访谈中的四级差异化定价模型，并为自己的课程新增了"经理人"培训组。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 读完一本好书但几个月后完全想不起核心内容
2. 有一堆读书笔记/文章摘录但每次要回顾时都必须从头通读
3. 准备写文章、做分享或做决策时，需要快速从大量笔记中提取关键信息
4. 感到"看了很多但记不住、用不上"
5. 想对现有的笔记库做一次价值提升，但不确定应该投入多少时间

### 语言信号 (用户的话里出现这些就应激活)

- "读完就忘了"
- "笔记太长看不下去"
- "怎么画重点/做笔记才有效"
- "笔记回顾太费时间"
- "怎么快速提取一本书的精华"
- "阅读效率低"

### 与相邻 skill 的区分

- 与 `second-brain-code` 的区别：渐进式归纳法是CODE中"提炼"环节的具体技术
- 与 `capture-criteria` 的区别：抓取标准管"记什么"，渐进式归纳法管"记完后怎么加工"
- 与 `intermediate-packets` 的区别：渐进式归纳法的产物（高亮笔记、纲要）是最常用的半熟素材类型之一

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **选一篇最近保存但还没加工过的笔记**
   - 让用户打开笔记应用，找一条最近抓取但从未回顾过的内容
   - 完成标准：选定了一篇目标笔记

2. **执行第1层提炼：精简原始摘录**
   - 如果原文摘录超过原文10%，引导用户删减至10%以内
   - 完成标准：摘录内容为精选段落，而非全文拷贝

3. **执行第2层提炼：加粗核心要点**
   - 让用户通读摘录，将关键词、关键句加粗
   - 引导："只标记那些让你心跳加速的内容。不要理性分析，追随直觉。"
   - 量化标准：加粗内容不超过整篇笔记的10-20%
   - 完成标准：仅浏览加粗部分就可以掌握笔记大意

4. **执行第3层提炼：高亮终极精华（可选）**
   - 仅聚焦加粗部分，将其中"百里挑一"的内容高亮
   - 引导："如果这篇笔记只剩一句话能留下来，是哪句？"
   - 完成标准：高亮内容不超过加粗内容的10-20%
   - 判停条件：如果笔记不够重要或用户时间有限→跳过此步

5. **验证"可见性"**
   - 让用户只看加粗和高亮内容，30秒内判断能否理解这篇笔记的核心
   - 如果能→提炼成功。如果必须读原文才能懂→标记不够精准，需要调整
   - 完成标准：用户在30秒内重拾了笔记的要点

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 初次阅读时就想做到完美标注——这会导致"过度标记"（什么都觉得重要），反而让标注失效
- 没有明确输出目标时批量提炼所有笔记——这是"亏本生意"，投入时间与回报不成正比
- 需要完整保留原文精确性的法律文件、学术引文——这些场景下删减提炼有风险

### 作者在书中警告的失败模式

- **过度标记重点**：将80%以上的内容加粗或高亮，标注后和原文几乎一样长。根源是缺乏取舍的勇气——"如果什么都想要，最后就什么都得不到。"
- **漫无目的的提炼**：花一整个周末"整理笔记"但没有产出任何作品。提炼必须在"准备创作时"进行。
- **故作高深**：用理性分析每一个标注决策，而非听从直觉。应该追随内心的悸动——"哪些文字让你目不转睛、心跳加速？"

### 作者的盲点 / 时代局限

- 假设用户以文字型内容为主要信息消费形式，对播客、视频等非文字内容的提炼支持有限
- "10-20%"递减规则缺乏严格的实证依据，更多是经验法则

### 容易混淆的邻近方法论

- 传统的"画重点"学习法：一次性标注、服务于考试记忆，而非分层的、服务于长期复用。渐进式归纳法的关键差异在于"跨时间的层层叠加"和"按需深入"。
- SQ3R阅读法（浏览-提问-阅读-复述-复习）：面向理解性阅读，渐进式归纳法面向笔记的长期价值提升。

## 合理化防御（Prebuttal）

Agent 在压力下会为跳过渐进式归纳找到合理理由。以下是预先反驳：

| Agent 的借口 | 预先反驳 |
|-------------|---------|
| "文章很短，不需要提炼" | 长度不是提炼的标准——信息密度才是。一段 100 字的高密度洞见比 1000 字废话更需要提炼 |
| "做 L1 就够了，L2-L4 太费事" | L1 只是删减。真正的知识压缩发生在 L2（加粗）和 L3（高亮）。没有 L2-L3 的笔记 = 没有入口 |
| "第一次读，以后再补提炼" | "以后再补" = "永不补"。提炼要做就在第一次保存时做完，记忆不会变得更清晰 |
| "原文写得很好了，不需要删" | 原文写得好≠你的笔记需要全文。10% 原则不是建议，是铁律 |
| "核心要点可以省略，正文说得很清楚" | 核心要点是三个月后你回看这篇笔记的第一入口。没有它 = 没有索引 |
| "这是小修改，不用走完整流程" | 每条规则都在这类借口中被绕过。没有"小修改"豁免权 |
