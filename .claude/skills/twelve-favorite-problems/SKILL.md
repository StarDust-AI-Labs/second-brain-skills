# 十二个兴趣问题 (12 Favorite Problems)

## R — 原文 (Reading)

> "你需要在脑海中整理出一打最感兴趣的问题，尽管大多数的时间可能用不着它们。每次当你听说或者学习到某些新发现或新技术时，就用这十二个问题进行测试，看看能否对其中的一项或几项有所帮助，那么每隔一段时间便会获得一些突破。不明就里的人们则会惊叹：'这是怎么做到的？他一定是个天才！'"
>
> —— 理查德·费曼，转引自《打造第二大脑》第四章

> "兴趣问题的魅力在于它们是经久不衰的。虽然每个问题的具体表现形式可能会随着项目、工作、情感和职业的不同而有所变化，但它们的陪伴是长期的，而且历久弥新。"
>
> —— 《打造第二大脑》第四章

## I — 方法论骨架 (Interpretation)

十二个兴趣问题是一套**用好奇心来导航信息消费**的方法。它的核心逻辑是：不是所有的信息都值得你关注——只有那些能帮助你回答你真正关心的问题的信息，才算遇到了对的"收件人"。

**问题的特征**：
- **开放式的**：不能是"是/否"可以回答的。好的问题像"如何让团队会议减少50%的时间同时提升效果？"，不好的问题像"远程办公好吗？"
- **跨领域的**：不局限于你的专业——费曼的问题横跨物理学、邦戈鼓、绘画、旅行。跨领域的问题最能激发跨界联想。
- **长期陪伴的**：这些问题可能陪伴你数年甚至一生。它们的答案永远在进化。
- **源自真实好奇而非功利需求**：你小时候对什么着迷？什么东西让你忘记时间？这些问题往往比"如何升职加薪"更有生命力。

**使用方法**：
每当接触新信息时——读一篇文章、听一期播客、看一段视频、参加一场会议——把这条信息扔向你的一打问题："它能帮助我回答其中任何一个吗？"如果能，记录下来。如果不能，快速划过。

这不是一个"要解决"的任务清单。这些问题就像一排永远亮着的探照灯，扫过信息海洋时只照亮与你的深层好奇相关的内容。

## A1 — 书中的应用 (Past Application)

### 案例 1：费曼的餐盘顿悟
- **问题**：费曼始终带着一打问题生活。其中一些问题涉及物质的运动和相互作用规律。
- **方法论的使用**：在康奈尔大学学生餐厅，他看到有人抛起一个刻有校徽的餐盘——盘子一边旋转一边摇摆。他注意到徽章的变化揭示出旋转和摇摆的节奏不同步。他将这个观察扔向他的"问题网"——这两种周期运动之间是否存在数学关系？
- **结果**：他发现盘子的摇摆和旋转频率之比为2:1，这个"好玩"的发现后来引出了更深层次的物理研究，最终为他赢得了诺贝尔奖。

### 案例 2：作者自己的童年问题
- **问题**：作者小时候玩乐高积木的方式与其他孩子不同——他迷恋的是"如何从一片混沌中取得创新"这一问题。
- **方法论的使用**：他花费大量时间对积木进行排列重组，根据颜色、尺寸、主题开发不同的排列计划。他深信"如果能够构建起正确的系统性方法"，就能创造出原创作品。
- **结果**：这一问题时至今日仍在激励着他——只是表现形式从乐高积木变成了数字信息管理。"请不要寄希望于寻求一劳永逸的固定答案，而应当将这一问题作为指引我们学习方向的北极星。"

### 案例 3：费曼的多领域成就
- **问题**：费曼一生中保持着对物理、邦戈鼓、绘画、旅行、生物、计算机等多个领域的兴趣问题。
- **方法论的使用**：他的"一打问题"跨越了完全不同的学科。当他在某个领域学到新东西时，会测试它是否能应用于其他领域的问题。
- **结果**：费曼在理论物理学获诺贝尔奖、在巴西学会了表演级别的邦戈鼓和康加鼓、参与了挑战者号航天飞机空难调查。外界惊叹他的"天才"，而他将一切归功于"随身携带一打问题"的方法。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 每天刷1-2小时社交媒体/信息流，看完就忘，感觉时间被浪费
2. 对很多领域感兴趣但都不深入，"样样通样样松"
3. 想在信息消费中建立"过滤器"，从被动刷内容变成主动寻找答案
4. 职业生涯迷茫期——不确定自己真正关心什么、该学什么
5. 想要培养跨学科思维，但不知从何入手

### 语言信号 (用户的话里出现这些就应激活)

- "刷了很多内容但什么也没留下"
- "信息太多看不过来"
- "怎么找到自己的兴趣方向"
- "学习没有重点"
- "不知道学什么好"
- "每天花很多时间看东西但感觉在浪费时间"
- "如何筛选有用的信息"

### 与相邻 skill 的区分

- 与 `capture-criteria` 的区别：兴趣问题是"预设的好奇心方向"（战略层），抓取标准是"遇到信息时的筛选工具"（战术层）。两者配合使用。
- 与 `second-brain-code` 的区别：兴趣问题主要服务于CODE的"抓取"环节，提供一个"该抓什么"的方向感。
- 与 `diverge-converge` 的区别：兴趣问题帮助你在发散阶段不会迷失方向——好奇心就像指南针。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **挖掘用户的深层好奇**
   - 引导性提问：
     - "你小时候对什么最着迷？"
     - "什么话题让你和朋友聊天时忘记时间？"
     - "有没有什么问题，你已经思考了好几年但还没找到满意答案的？"
     - "如果有无限时间和资源，你想研究什么？"
   - 请用户向亲友询问自己童年时的重度爱好
   - 完成标准：用户写出了至少5个候选问题

2. **将候选问题改写为开放式问题**
   - 检查每个问题是否可以通过"是/否"回答→如果是，改写为"如何/为什么/在什么条件下"
   - 例子："远程办公好吗？"→改写为"在什么条件下，远程办公比办公室办公更有利于团队的创造力和凝聚力？"
   - 完成标准：所有候选问题都是开放式的

3. **整理出一份8-12个问题的清单**
   - 不要纠结凑足正好12个——8个也是可以的
   - 不要担心清单是否"完美"——这是第一版，后续可以持续迭代
   - 将清单保存为一条笔记，放在数字笔记最显眼的位置
   - 完成标准：清单已保存为笔记

4. **实践"用问题过滤一条信息"**
   - 让用户打开最近保存或正在阅读的一篇文章
   - 用清单中的问题逐一测试：这条信息能帮助我回答其中任何一个问题吗？
   - 如果能→在笔记中标注对应的问题编号；如果不能→问自己"那我为什么要读这篇文章？"
   - 完成标准：用户亲手完成了一次"问题过滤"

5. **定期迭代**
   - 建议每月回顾一次问题清单：哪些问题已经不再让你感兴趣了？有没有新问题冒出来？
   - 完成标准：用户知道如何维护这份清单

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户已经有非常清晰的研究方向和学习计划——问题清单可能成为多余的负担
- 用户只是想抱怨信息过载但无意改变消费习惯——这需要行为改变而不只是工具
- 纯任务驱动的信息需求（如"查一下明天天气"）——兴趣问题是为开放式探索设计的，不是为事实查询

### 作者在书中警告的失败模式

- **把问题当成待办事项**：兴趣问题不是要"解决"的任务清单，而是伴随终身的思考方向。它们没有deadline。
- **问题过于封闭**：是非题（如"我应该跳槽吗？"）无法激发持续探索和跨界联想。必须是开放式问题。

### 作者的盲点 / 时代局限

- 费曼的方法源自一个信息相对稀缺的时代（需要主动寻找信息），在信息冗余的算法推荐时代，用户需要更强的自律来用问题抗衡算法
- 方法偏向直觉驱动，对于习惯于结构化目标管理的人可能需要适应期

### 容易混淆的邻近方法论

- IKIGAI（日本人的人生意义圆圈）：关注的是找到人生使命的四个维度交集。兴趣问题更轻量、更灵活——你不需要找到"唯一的目标"，只需要一打好奇心。
- 研究课题列表（Research Questions）：学术研究课题通常需要严谨的方法论和产出，兴趣问题可以是松散的、探索性的。
