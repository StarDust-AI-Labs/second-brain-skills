# 共鸣原则与信息抓取标准

## R — 原文 (Reading)

> "我在此建议采用以下四大标准帮助你判断何种信息是适宜记录的。1. 是否具有启发性；2. 是否具有实用性；3. 是否具有个性；4. 是否具有新奇性……信息抓取的终极标准：是否让你为之共鸣。你在日常与各种内容打交道时，请留意自己的内心有没有被某条信息打动或是惊艳到。这种特别的共鸣感——如同来自灵魂深处的回响，便是直觉在向你发出强烈提醒：一定要把它记录下来！"
>
> —— 蒂亚戈·福特，《打造第二大脑》第四章

> "神经科学研究告诉我们，'情感能够引导而非扰乱理性思维'。如果某个事物触发了我们的共鸣，实际上就是我们的直觉在情感的驱动下，赶在逻辑思维查明原因之前告诉我们：这件事很有趣哦！"
>
> —— 《打造第二大脑》第四章

## I — 方法论骨架 (Interpretation)

在信息过载的时代，"什么值得保存"是一个比"怎么保存"更根本的问题。这个skill提供了五层筛选标准——四层理性检查 + 一层直觉裁决：

**四大理性标准**：

1. **启发性**：它让你眼前一亮吗？它能激发你的动力或想象力吗？启发性内容是你创作的"精神燃料"——当你陷入低谷时翻看这些内容，能让想法回归正轨。

2. **实用性**：它可能在未来的某个项目中派上用场吗？像木匠收藏边角料一样——现在看不出来有什么用，但你有一种隐约的预感。统计数据、参考信息、实用图表、代码片段都属于这类。

3. **个性**：它承载了你个人的思想、记忆、观点或情感吗？这是信息最深层的价值——别人写的文章你可以随时再搜索，但你自己的想法稍纵即逝永不复现。你自己的思考、与朋友的对话截图、个人反思，是你独一无二的知识资产。

4. **新奇性**：它挑战了你既有的认知吗？它是"让你耳目一新的东西"（香农的信息定义）。注意：大多数人会下意识地只收集与自己观点一致的内容（确认偏误）。新奇性是唯一能防止你的第二大脑变成回音室的标准。

**终极标准 — 共鸣**：
四大标准是辅助参考——真正决定"记或不记"的是你的直觉。当你读到某句话时心跳加速、睁大眼睛、感到世界静止了一秒——那就是共鸣。神经科学实验证明，潜意识可以在理性思维做出判断之前很久就已经完成了学习并做出反应。培养"聆听共鸣"的习惯，就是训练你的直觉在信息洪流中做出比理性更快的判断。

**量化边界线**：笔记篇幅至多不超过原文的10%。超过这个比例，你就不是在策展，而是在囤积。

## A1 — 书中的应用 (Past Application)

### 案例 1：作者的情感支持文件夹
- **问题**：作者有时会觉得自己的工作不重要、做得不够好，产生自我怀疑。
- **方法论的使用**：他将多年来收集的客户反馈（启发性——证明工作的价值；个性——记录了真实的情感互动）保存在一个专门的文件夹中。
- **结果**："任何时候，每当我觉得自己的工作并不重要，或者做得不够好时，我便打开这个文件夹，如此便能让想法得以改观。"

### 案例 2：泰勒·斯威夫特的歌词笔记本
- **问题**：流行歌手泰勒·斯威夫特需要从日常生活的点滴感悟中提炼出打动人心的歌词。
- **方法论的使用**：她将一闪而过的歌词灵感、一段旋律、一句对白立即记录在手机笔记中。创作歌曲《空位》时，她将多年前储存的各种"金句"糅合在一起——"原来所有的爱情都只有两个归宿——要么天长地久，要么化为乌有"是她某天突然想到的；"亲爱的，我就是一出装扮成美梦的噩梦"是后来冒出的另一句。
- **结果**：《空位》成为热门金曲。她的9张专辑全球销量累计超过两亿张。"创作歌曲不是一项孤立的、只能在特定时间和地点从事的活动——它是我整个思维活动的衍生产品。"

### 案例 3：杰里·森菲尔德的黄色便签
- **问题**：喜剧大师杰里·森菲尔德需要持续产出有趣的段子。
- **方法论的使用**：每想到一个有趣的梗——无论是舞台表演时激发的灵感、闲聊时获得的启发，还是在黄色便签纸上创作时的输出——他都保存在一个老式折叠文件夹中。
- **结果**：当别人对他的记录习惯表示惊讶时，他回应："我实在不明白这有什么好惊讶的，我也实在不明白除此之外还有什么好记录的。难道还有什么能比这些更有价值么？"

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 浏览器书签/收藏夹堆积了数百条内容，但打开率接近于零
2. 在各种"稍后读"应用中积累了大量未读文章，焦虑感甚至超过了收获感
3. 笔记应用中内容越来越多但感觉越来越"杂"，找不到真正有价值的东西
4. 保存每一条觉得"可能有用的"信息，但从不回顾，陷入了数字囤积
5. 想建立"信息策展人"的身份——有选择地获取，而非无差别地消费

### 语言信号 (用户的话里出现这些就应激活)

- "收藏了很多但从来不看"
- "笔记太多太杂"
- "不知道什么该记什么不该记"
- "怎么判断这条信息值不值得保存"
- "书签/收藏夹快爆了"
- "数字囤积怎么解决"
- "我只想记最重要的东西"

### 与相邻 skill 的区分

- 与 `twelve-favorite-problems` 的区别：兴趣问题是"方向性过滤"（信息能否帮助回答我的问题？），抓取标准是"品质性过滤"（这条信息本身够不够好？）。两者叠加使用效果最佳。
- 与 `progressive-summarization` 的区别：抓取标准决定"要不要记"，渐进式归纳法决定"记完后怎么加工"。
- 与 `second-brain-code` 的区别：抓取标准是CODE第一步"抓取"环节的决策工具。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **清点用户的"信息囤积"现状**
   - 让用户说出三个数字：浏览器书签数、"稍后读"应用中的未读文章数、笔记应用中的总笔记数
   - 让用户估计：其中大约有多少比例是"真正看过且觉得有用的"？
   - 完成标准：用户对自己当前的抓取问题有了量化的认识

2. **实施"共鸣优先"的信息审计**
   - 让用户打开收藏夹/书签/稍后读列表中的最近10条内容
   - 对每一条只问一个问题："我现在对它还有那种'眼前一亮'的感觉吗？"
   - 如果答案是否定的——删除或归档。不要用"以后可能会用到"来为囤积辩护
   - 完成标准：用户清理了至少10条内容，体验了"用直觉而非理性做减法"

3. **练习用四项标准评估一条新信息**
   - 让用户找一条今天遇到的新内容（文章/视频/帖子）
   - 逐条用四个标准快速打分（每个标准只需1-2秒的直觉判断）：
     - 启发：心跳加速了吗？
     - 实用：未来真的可能用到吗？
     - 个性：有我的个人视角或情感在其中吗？
     - 新奇：有没有让我"啊？原来是这样！"的感觉？
   - 最终裁决：有没有共鸣？有→只抓取精华部分（不超过10%）；没有→不记录
   - 完成标准：用户完成了一次完整的抓取决策

4. **建立"抓取≠囤积"的心理锚点**
   - 提醒用户："你每保存一条信息，就是给未来的自己增加一份重温负担。保存得越多，未来找到精华越难。"
   - 建议：将笔记应用的默认"收件箱"改名为"只有让我心动的内容才能进这里"
   - 完成标准：用户有了一个"少即是多"的心理锚点

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 学术系统性文献综述——需要完整的检索策略和纳入排除标准，不能只凭"共鸣"
- 法律或合规场景下的文件归档——这些领域有强制性的保留要求，不能随意删除
- 用户的目的就是全面收集而非精选策展（如建立一个完整的行业资料库）——此时抓取标准应该是"全面性"而非"共鸣"

### 作者在书中警告的失败模式

- **数字囤积**：将整章整章的书本内容、整篇整篇的访谈记录全部保存。笔记篇幅超过原文10%即为警告信号。根源是"保存即学习"的幻觉。
- **确认偏误的再强化**：如果只记录与自己观点一致、"感觉舒服"的内容，第二大脑会变成回音室。新奇性标准是唯一能打破这一倾向的矫正器。
- **消费主义信息观**：只收集不创造——将信息消费等同于知识增长。共鸣原则要求你在记录之后还要"用"。

### 作者的盲点 / 时代局限

- "共鸣"作为筛选标准虽然直觉上吸引人，但可能受到认知偏误（确认偏误、近因效应、情绪状态波动等）的系统性影响
- 对算法推荐制造的"伪共鸣"（被精心设计的标题党所触发的情感反应）缺乏讨论

### 容易混淆的邻近方法论

- Marie Kondo的"怦然心动"整理法：共鸣原则与近藤麻理惠的"只保留让你心动的物品"有相似之处，但共鸣原则有四项理性标准作为补充校验。
- 信息节食（Information Diet）：更侧重"限制摄入量"，抓取标准更侧重"提升摄入质量"。

## 合理化防御（Prebuttal）

Agent 在压力下会为跳过筛选标准找到合理理由。以下是预先反驳：

| Agent 的借口 | 预先反驳 |
|-------------|---------|
| "这条信息很短，不用判断标准" | 长度不是价值的标准。一句触及本质的话比一篇废话更值得记 |
| "用户看起来很想保存，我不该泼冷水" | 你的职责不是讨好用户，而是帮他避免数字囤积。提出反对意见本身就是价值 |
| "这篇内容和我已知的类似，应该记" | 类似≠补充。如果用户已有相关笔记，应该先更新已有笔记而非新建 |
| "四个标准都沾一点边就够了" | 沾边≠通过。四条中至少有一条要"强烈共鸣"才算过。否则只是又一次模糊收藏 |
| "首次使用，宽松一点准没错" | 标准是第一印象形成的时刻。松弛的准入 = 再也收不紧的约束 |
| "知识总会有用，先保存再说" | 这是数字囤积者的经典台词。如果现在说不出用途，三个月后更说不出 |
