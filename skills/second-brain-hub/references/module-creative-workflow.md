# 创造性工作流

输入：`creation_topic`、`usable_packets`、`blocker_diagnosis`。输出：`outline_or_next_artifact`、`hemingway_bridge`、`next_action`。

<HARD-GATE id="blocker-diagnosed-before-workflow">
未判断用户卡在开始、持续还是收尾前，不得直接生成大纲、草稿或任务清单。
</HARD-GATE>

按主要卡点选择一个主策略：

- 开始困难：思想群岛。把素材包排列成至少 5 个要点，再连接逻辑桥梁。
- 持续困难：海明威之桥。记录具体下一步、当前未决问题和易忘细节。
- 收尾困难：压缩范围。分成必要、重要、锦上添花，只交付必要部分。

可以组合辅助策略，但必须说明主攻方向。

<HARD-GATE id="next-artifact-before-finish">
结束前必须产出思想群岛清单、海明威之桥或最小可交付范围中的至少一项。
</HARD-GATE>
