# 用户可见输出卡片

内部台账默认不展示。根据阶段使用以下最小格式。

## 开始

```text
【开始】我会按「{场景}」流程处理
【会做】{3-5 个动作，用 → 串联}
【需要你确认】{一个问题；无需确认时写“暂时不需要”}
```

## 进度

```text
【进度】{当前}/{总数}：{动作}
【发现】{关键发现，可省略}
【下一步】{下一动作}
```

## 决策

```text
【需要你决定】{一个明确问题}
【我的建议】{推荐选项及理由}
【可选项】{A / B / C}
```

## 完成

```text
【完成】{结果}
【位置】{Vault 内路径；只读或诊断场景可省略}
【核心要点】{1-3 条}
【处理摘要】{来源、数量、提炼层级或归属}
【下一步】{最自然的后续动作}
【流程校验】{用普通语言说明关键门控}
```

## 首次成功

仅在 `onboarding.completed=true` 且 `onboarding.examples_shown=false` 的第一次成功写入后使用：

```text
【已完成】你的第二大脑已经可以使用，第一条内容已保存
【位置】{实际路径}
【接下来可以试试】
- “保存这篇文章 + URL”
- “找一下关于 {当前主题} 的笔记”
- “把刚才的笔记提炼到 L2”
```

输出后将 `first_success_at` 写为当前时间，并将 `examples_shown=true`。后续恢复使用普通最小结果卡片。

路径、文件名、数量和是否实际写入必须明确。不要显示裸命令、内部字段名或大段 JSON。
