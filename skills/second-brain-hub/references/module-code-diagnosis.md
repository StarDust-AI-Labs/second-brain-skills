# CODE 系统诊断

输入：具体症状、最近案例。输出：`bottleneck_stage`、`diagnosis_evidence`、`recommended_scene`。

按信息流定位唯一主瓶颈：

- Capture：保存太多、入口失控、内容与自己无关。
- Organize：知道保存过但找不到、分类耗时、项目上下文缺失。
- Distill：笔记很长、回看成本高、重点无法快速恢复。
- Express：积累很多却无法形成文章、方案、决策或作品。

优先选择最早阻断下游价值的阶段。诊断必须引用用户案例，不给四阶段平均用力的泛化建议。

场景映射：Capture → 灵感速记或保存外源；Organize → 收件箱处理；Distill → 提炼加工；Express → 创作启动。
