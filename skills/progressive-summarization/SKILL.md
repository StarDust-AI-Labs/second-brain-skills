# 渐进式归纳法 (Progressive Summarization)

## I — 方法论骨架 (Interpretation)

渐进式归纳法是一套**按需深入**的笔记提炼系统——随着每次"重逢"逐步加深提炼。

四层提炼（快速清单）：

1. **第1层 — 原始摘录（"土壤"）**：原文精华，≤10%
2. **第2层 — 加粗要点（"原油"）**：核心关键词加粗，≤L1 的 10-20%
3. **第3层 — 高亮精华（"黄金"）**：百里挑一的内容高亮，≤L2 的 10-20%
4. **第4层 — 纲要总结（"宝石"）**：用自己的话写纲要，仅极少数笔记

> 四层提炼的详细解释与关键原则，参见 [four-layers-deep-dive.md](references/four-layers-deep-dive.md)

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 读完一本好书但几个月后完全想不起核心内容
2. 有一堆读书笔记/文章摘录但每次要回顾时都必须从头通读
3. 准备写文章、做分享或做决策时，需要快速从大量笔记中提取关键信息
4. 感到"看了很多但记不住、用不上"
5. 想对现有的笔记库做一次价值提升，但不确定应该投入多少时间

### 语言信号 (用户的话里出现这些就应激活)

- "读完就忘了"
- "笔记太长看不下去"
- "怎么画重点/做笔记才有效"
- "笔记回顾太费时间"
- "怎么快速提取一本书的精华"
- "阅读效率低"

### 与相邻 skill 的区分

- 与 `second-brain-code` 的区别：渐进式归纳法是CODE中"提炼"环节的具体技术
- 与 `capture-criteria` 的区别：抓取标准管"记什么"，渐进式归纳法管"记完后怎么加工"
- 与 `intermediate-packets` 的区别：渐进式归纳法的产物（高亮笔记、纲要）是最常用的半熟素材类型之一

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

<HARD-GATE id="source-selected-before-distill">
未确定目标笔记或原文内容之前，不得开始提炼或改写。

必须先确认:
1. 处理对象是哪一篇笔记或哪段内容
2. 当前已有提炼层级
3. 本次目标层级是 L1、L2、L3 或 L4

禁止的行为:
- 没读内容就生成总结
- 同时处理大量笔记却不设优先级
- 将搜索结果当作已提炼内容
</HARD-GATE>

<HARD-GATE id="density-limit-before-update">
写回笔记前，必须检查本层提炼比例是否符合限制。

比例约束:
1. L1 摘录不超过原文 10%
2. L2 加粗不超过 L1 的 10%-20%
3. L3 高亮不超过 L2 的 10%-20%

禁止的行为:
- 全文加粗或大段高亮
- 用"都很重要"绕过取舍
- 没有核心要点就结束更新
</HARD-GATE>

1. **选一篇最近保存但还没加工过的笔记**
   - 让用户打开笔记应用，找一条最近抓取但从未回顾过的内容
   - 完成标准：选定了一篇目标笔记

2. **执行第1层提炼：精简原始摘录**
   - 如果原文摘录超过原文10%，引导用户删减至10%以内
   - 完成标准：摘录内容为精选段落，而非全文拷贝

3. **执行第2层提炼：加粗核心要点**
   - 让用户通读摘录，将关键词、关键句加粗
   - 引导："只标记那些让你心跳加速的内容。不要理性分析，追随直觉。"
   - 量化标准：加粗内容不超过整篇笔记的10-20%
   - 完成标准：仅浏览加粗部分就可以掌握笔记大意

4. **执行第3层提炼：高亮终极精华（可选）**
   - 仅聚焦加粗部分，将其中"百里挑一"的内容高亮
   - 引导："如果这篇笔记只剩一句话能留下来，是哪句？"
   - 完成标准：高亮内容不超过加粗内容的10-20%
   - 判停条件：如果笔记不够重要或用户时间有限→跳过此步

5. **验证"可见性"**
   - 让用户只看加粗和高亮内容，30秒内判断能否理解这篇笔记的核心
   - 如果能→提炼成功。如果必须读原文才能懂→标记不够精准，需要调整
   - 完成标准：用户在30秒内重拾了笔记的要点

## 合理化防御（Prebuttal）

Agent 在压力下会为跳过渐进式归纳找到合理理由。以下是预先反驳：

| Agent 的借口 | 预先反驳 |
|-------------|---------|
| "文章很短，不需要提炼" | 长度不是提炼的标准——信息密度才是。一段 100 字的高密度洞见比 1000 字废话更需要提炼 |
| "做 L1 就够了，L2-L4 太费事" | L1 只是删减。真正的知识压缩发生在 L2（加粗）和 L3（高亮）。没有 L2-L3 的笔记 = 没有入口 |
| "第一次读，以后再补提炼" | "以后再补" = "永不补"。提炼要做就在第一次保存时做完，记忆不会变得更清晰 |
| "原文写得很好了，不需要删" | 原文写得好≠你的笔记需要全文。10% 原则不是建议，是铁律 |
| "核心要点可以省略，正文说得很清楚" | 核心要点是三个月后你回看这篇笔记的第一入口。没有它 = 没有索引 |
| "这是小修改，不用走完整流程" | 每条规则都在这类借口中被绕过。没有"小修改"豁免权 |
