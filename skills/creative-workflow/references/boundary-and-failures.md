# 边界与失败模式

## 不要在以下情况使用此 skill

- 简单重复性任务（填表、签到、常规审批）——这些不需要创意工作流
- 用户已经接受了严格的敏捷/Scrum方法论且有Scrum Master支持——压缩范围的理念已内嵌其中
- 用户对创作过程没有持续性问题——只是在某一天状态不好，这是正常波动

## 作者在书中警告的失败模式

- **FOMO（错失恐惧症）**：在思想群岛阶段，总觉得还应该再收集更多素材才能开始。提醒：你是在做雏形，不是最终作品。
- **"时机不成熟"拖延**：把压缩范围当作"不用交付"的借口。压缩范围的目的是交付更小的版本，而非无限期推迟。
- **海明威之桥写成流水账**：不是记录"今天做了什么"，而是记录"下次该做什么+当前状态+易忘细节"。

## 作者的盲点 / 时代局限

- 偏向独立创作者的工作模式，团队协作中的集体创意流程（如小组头脑风暴的过渡机制）讨论不足
- 海明威的写作策略适合长篇创作，对于社交媒体时代的微内容创作（一条推文、一张图）的适用性需要重新评估

## 容易混淆的邻近方法论

- 思维导图 (Mind Mapping)：思维导图是发散工具，思想群岛同时服务于发散和聚合——它既是素材的集合，也是逻辑的框架。
- Pomodoro番茄工作法：番茄工作法管理的是"注意力持续时间"，海明威之桥管理的是"跨工作阶段的意图连续性"。
